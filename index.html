Mars Rover Docs

These are very simple docs to help you manage this Mars Rover app

How to use it

  • Step 1: Upload instructions (It must be a .txt file)
  • Step 2: Start simulation
  • Step 3: (optional) Download output txt file
  • Step 4: Start a new simulation if you want to
****************************

Be careful!

The input .txt file should follow some basic rules:

  • The 2nd line must start with "Size".
    (x,y) info should be separated by a space *To avoid problems with the ui, the current application supports these max values: (7,6)
  • Obstacle lines must start with "Obstacle" and (x,y) should be separated by a space* *In case of invalid input [(x,y) greater than grid, invalid (x,y)] the obstacle will be ignored and not showed on the map
  • Command lines must start with one of accepted command (L, R, F, B) and should contain just accepted commands: Invalid commands will be ignored by the rover

For more information please refer to the instructions.txt file included with the application.

Instruction .txt example

Grid
Size 5 4
Obstacle 2 0
Obstacle 0 3
Obstacle 3 2
Commands
RFF
RF
LFRFFLFFFLL
